Meine Hacks

Ich suche und melde Sicherheitslücken als Hobby, damit die Lücken geschlossen werden und das Intenet ein Stück sicherer wird. Die Betreiber wurden von mir (oder dem BSI) über die Sicherheitsprobleme informiert.

Ich teste nur ein "Proof of Concept" (= Machbarkeitsnachweis) und nicht mehr. Damit ich nicht selbst nachprüfen muss, ob die Lücken auch wirklich geschlossen wurden, sende ich meine Hinweise auch an das BSI (Bundesamt für Sicherheit in der Informationstechnik).

Anonyme Hinweise an das BSI können mit diesem Webformular versendet werden.

neu

  • Warnung vor kritischen Sicherheitslücken im "Deutschland-Scan". Ein Hobby-Projekt, um die gesamte deutsche, öffentliche Infrastruktur automatisch auf Sicherheitslücken zu scannen. Es wurden über 2.000 Bundesbehörden, Landkreise, Verwaltungen, Schulen und Städten in ganz Deutschland getestet und dabei 59.000 Webserver untersucht.
    Ingesamt wurden über 70 kritische Sicherheitslücken an das BSI gemeldet und behoben. Sicherheitslücken, die als kritisch eingestuft werden, können einem Angreifer z.B. Zugriff auf den Server geben, sodass interne Daten ungeschützt waren.
  • Bundesjustizamt: Kritische RCE (Remote Code Execution)
    CVE-2021-26084: "OGNL injection vulnerability exists that would allow an unauthenticated attacker to execute arbitrary code on a Confluence Server or Data Center instance"
  • Critical server-side request forgery (SSRF) vulnerability.
    CVE-2021-40438: "mod_proxy to forward the request to an origin server choosen by the remote user"
    Finanzamt Baden-Württemberg, Breitbandseite Bayern, Bundesamt für Materialforschung, Videokonferenzen von Niedersachsen, Bibliothek der Helmut Schmidt Universität, Kommunales Informationssystem in Ludwigsburg, Stadtverwaltung Baden-Baden, Stadt Bocholt, Stadt Aschaffenburg, ...
    Nach meiner Meldung wurden vom CERT-Bund bereits gehackte Server vorgefunden.
  • Directory traversal, read and delete access to sensitive files
    CVE-2020-3187: "allow an unauthenticated, remote attacker to conduct directory traversal attacks and obtain read and delete access to sensitive files on a targeted system."
    Bundesministerium für Digitales und Verkehr, Deutscher Wetterdienst, Deutschen Nationalbibliothek, im VPN der SWU Stadtwerke Ulm, Forst Lausitz, Essener Arbeitsgesellschaft.
  • SQL Injection in JCK Editor in Joomla (CVE-2018-17254) bei der Stadt Dessau-Rosslau.
  • Weak encryption flaw resulting in remote code execution (CVE-2017-1000486) bei der Stadt Burglengenfeld, Straßendatenbank Bochum.
  • ThinkPHP, kritische RCE (Remote Code Execution): Lernstandserhebungen an Hamburger Schulen.
  • Ungepatchte Exchange-Server mit kritischer Sicherheitslücke
    CVE-2021-34473: "Microsoft Exchange Server Remote Code Execution Vulnerability"
    Volkshochschule in Amberg, Landkreis Merzig-Wadern und mehreren Stadtverwaltungen: Bernsdorf, Bleckede, Dachau, Erkner, Heilbald Heiligenstadt, Klötze, Mölln, Plauen, Rendsburg, Sassnitz, Stadtbergen, Sulzbach Saar, Vellmar und im Exchange-Server im Theater in Freiburg, ... Heise Online Artikel
  • Proxyshell-Lücke (CVE-2021-34473) 1 von 3 möglicherweise nicht vollständig behoben: "Microsoft Exchange Server Autodiscover Server Side Request Forgery Authentication Bypass Vulnerability"

    Selbst auf manchen gepatchen Systemen (Version: 15.2.986.13, Version: 15.1.2308.20) ist es möglich über eine spezielle URL an interne Exchange-Seiten zu kommen. Mir ist aufgefallen, dass dies nur zu bestimmten Zeitpunkten möglich ist, also möglicherweise nur gleichzeitig mit einer Aktion eines Nutzer im Outlook Webclient auftritt.
    Das führt dazu, dass mindestens der interne Anmeldenamen geleakt wird und möglicherweise weitere interne Exploit-Ketten zu bestimmten Zeitpunkten möglich sind.
