Document Title:

La Poste FR Website - Local File Include Vulnerability



Release Date:
=============
2011-09-10


Vulnerability Laboratory ID (VL-ID):
====================================
266


Product & Service Introduction:
===============================
La Poste is the mail service of France, which also operates postal services in the French Overseas Departments 
of Réunion, Guadeloupe, Martinique and French Guiana, and the territorial collectivities of Saint Pierre and 
Miquelon and Mayotte. It is headquartered in the 15th arrondissement of Paris.

La Poste postage stamps are valid for use in the Overseas Departments, but not in the Territorial Collectivities, 
which issue their own stamps, separate from metropolitan France. It also operates the postal service in Monaco, 
while in Andorra, it operates postal services alongside those of Spain.

In addition to postal services, it also offers banking services (with La Banque Postale) and, via Chronopost, 
courier services. After the government, La Poste is the second biggest employer in France. It also offers webmail, 
providing @laposte.net email addresses and hosts approximately 1.6 million active email accounts.  La Poste owns 
German parcel delivery company Deutscher Paket Dienst (DPD).

(Copy of the Website: http://en.wikipedia.org/wiki/La_Poste_%28France%29 )


Abstract Advisory Information:
==============================
Vulnerability-Lab Team (A-HEALL) discovered a local file inclusion on the La Poste (FR) vendor website.

Affected Product(s):

=======================
Remote


Severity Level:
===============
Critical


Technical Details & Description:
================================
Local file inclusion is detected on the LaPoste website. The bug allows remote attackers to include (request) a local file of the server.


Vulnerable Module(s):
                                                                 [+] freeEditorial - (?page)

Pictures:
                                                                 ../1.png


Proof of Concept (PoC):
=======================
The vulnerability can be exploited by remote attackers. For example or reproduce ...

PoC: 
http://timbres.laposte.fr/freeEditorial.jgi?page=../../images/bg_leftbox.gif


Solution - Fix & Patch:
=======================
Restrict & parse the path request of ?page=  to disallow local/remote file requests.

Example:

if($_GET[file] == site) {
	include($_GET[page]);}
else {
	include(content.php);}
  


Security Risk:
==============
The security risk of the  local file include vulnerability is estimated as critical.
